═══════════════════════════════════════════════════════════════════════════════
WHAT THE ENDPOINT DOES
═══════════════════════════════════════════════════════════════════════════════

The endpoint:

1. READS SESSION FILES
   - Locates JSONL session file from agent ID
   - Path: C:\Users\abuck\.openclaw\agents\main\sessions\{sessionId}.jsonl

2. PARSES JSONL
   - Reads and parses one JSON object per line
   - Skips malformed lines gracefully
   - Continues parsing on errors

3. EXTRACTS ACTIVITY DATA
   - Current task (most recent user message, max 500 chars)
   - Status (working/idle/blocked/offline based on timestamps)
   - Progress estimate (from recent tool calls and errors)
   - Skills used (all deduplicated tool names)
   - Start time (when task began)
   - Last updated (most recent event)
   - Message count (total messages)
   - Entry count (total JSONL entries)

4. RETURNS JSON RESPONSE
   - Type-safe SubagentActivityRecord
   - All fields properly typed and validated
   - Comprehensive error information if needed

STATUS VALUES
═══════════════════════════════════════════════════════════════════════════════

working  → Active within last 90 seconds (🟢 Green)
idle     → Active 90s - 30m ago (🟡 Amber)
blocked  → Error or aborted (🔴 Red)
offline  → No activity for >30m or file not found (⚪ Gray)
